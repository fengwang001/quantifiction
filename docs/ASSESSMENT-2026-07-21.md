# 系统阶段性评估与修改建议

> 日期：2026-07-21 · 数据窗口：影子对比运行 ~112 分钟，156 笔成交

## 一、总体判断

**基础设施：优秀。策略表现：全部证伪（正是预期的、有价值的结果）。**

- 系统端到端跑通并经真实模拟盘验证：欧易接入（签名/下单/止损/对账）、实时信号、影子多策略对比、持久化、Agent 认知层、可视化看板。
- 6 个策略**全部净亏损**，但这是**零真金成本的证伪**——这正是影子模式的目的（宪法 VI「先验证后加码」「能证伪就别用真金」）。

## 二、表现数据与根因

| 现象 | 数据 | 解读 |
|---|---|---|
| 全部净亏 | 净利 -1.6 ~ -3.3 USD | 无一策略扣费为正 |
| **手续费是杀手** | 毛利≈0（±0.4），费 -1.8~-3.3 | 费用是毛利的 10-100 倍 |
| 胜率极低 | 0-11% | TP 几乎不触发，多数超时平仓 |
| 夏普深负 | -1.1 ~ -1.9 | 无统计意义上的 edge |

**根因（早已用数学预测）**：taker 往返 0.10% 手续费，TP 仅 0.25% → 需 ~80% 胜率才净赚，实际胜率 0-11%。**裸盘口信号的 edge 覆盖不了手续费。**

**Agent 增强的观察**：净利倒数第二（-2.15），0% 胜率。但样本仅 20 笔、仅 10 次辩论，且 agent stance 一直偏弱/中性（当前 -0.20），基本没改变底层 OBI 行为——**继承了 OBI 的手续费病，还没证明 agent 无用，但当前融合方式没救回来**。

## 三、修改建议（按优先级）

### P0 —— 手续费是第一问题，三个杠杆

1. **降频 + 拉长持仓 + 放大目标**：TP 从 0.25% 提到 1-2%，持仓小时级。手续费占比从"吃光"降到"零头"。代价：策略性质从高频变波段（但这本就是 $1000 该做的，见 PRD）。
2. **改用 maker 挂单（限价）**：欧易 maker 费 0.02%（甚至返佣），往返 0.04% vs taker 0.10%。代价：可能不成交，需挂单管理。
3. **只做高确信度机会**：交易频率降 5-10 倍，只在信号极强时入场。少交易=少费用+只在有 edge 时下注。

> 我的建议：**先做 1+3**（低频 + 强过滤），最容易验证、最贴合 $1000 定位。

### P1 —— 出场逻辑重构

胜率 0-11% 说明**出场是坏的**：TP 几乎不触发、全靠超时。建议：
- 用**追踪止损**或**信号反转出场**替代固定 TP
- 或"盈亏比不对称"：小止损、让盈利奔跑

### P2 —— 原策略快照 + 历史留痕（你的明确要求）

**现状**：成交留痕有（shadow_persist.json + agent_decisions.jsonl），但**策略定义本身没版本化**。

**建议实现**：
1. **策略快照**：每个策略定义（信号/阈值/TP/SL/时长）+ 内容哈希，冻结存档为不可变基线。人工建的策略打 `author=human`、`immutable=true`。
2. **版本关联**：每笔成交关联 `strategy_version_id`（哈希）——将来 agent 改/生成策略时，能追溯"哪个版本的策略产生了哪笔交易"。
3. **周期归档**：每日快照 shadow_persist + 策略定义到 `data/snapshots/{date}/`，可看演化。

### P3 —— Agent 迭代（之前提到的两条）

**第一步（近期，零风险）：Agent 生成策略（声明式 DSL）→ 生在影子 → provenance 权限**
- 既然人工的 6 个策略全废，让 agent **在白名单积木内组合生成新策略**（信号+算子+参数范围，非任意代码）
- agent 生成的策略自动进 shadow，`author=agent`，**agent 只能改自己建的，人工建的只读**（你提的原则）
- 扣费跑够 50+ 笔、净利为正，才提议提升——提升需闸门/人工确认
- **不碰宪法 II**（影子不下真单），是当前框架的自然延伸

**第二步（验证有效后）：正式接入 vendor/TradingAgents**
- 当前 agent 是我从零写的极简版（1 次调用），非 TradingAgents 代码
- TradingAgents 面向美股、多 agent 多轮辩论、一次决策十几次调用——**在当前 grsai 供应商（并发限制低、gpt-5.4 大量失败）上跑不动**
- 接入前提：① 简化版已证明 agent 有价值 ② 换并发宽松的 LLM 通道 ③ 数据源改造为加密

## 四、给决策者的一句话

**系统本身是成功的**（它能安全交易、能证伪烂策略、零成本）。**当前策略全废是好事**——它在真金之前就告诉你"这些不行"。下一步不是急着上真金，而是：**攻克手续费（低频+强过滤+maker）→ 让 agent 在影子里搜索策略空间 → 谁扣费能赚，谁才有资格碰真金。**

## 五、建议的执行顺序

1. 立即：**策略快照 + 历史留痕**（保护基线，为 agent 生成策略铺路）
2. 然后：**低频/maker 策略变体**加入影子对比（攻手续费）
3. 然后：**Agent 生成策略（DSL + provenance）**，让它在影子里搜
4. 观察：谁扣费为正 → 才讨论提升到 live（那时才涉及宪法修订）
